# FlowingFiles — WPF to React Conversion Plan

## Context

FlowingFiles is a WPF desktop app that helps organize Brazilian accounting/fiscal documents monthly. Users select files, verify completeness via color-coded status indicators, and export them as a structured ZIP. The goal is to convert this into a **React web application** that runs entirely in the browser (no backend). The export will be **ZIP download only** (no folder export). AI classification is out of scope for now.

## Tech Stack

- **React 18** with **TypeScript**
- **Vite** as build tool
- **CSS Modules** for styling
- **JSZip** for ZIP generation
- **file-saver** for triggering downloads
- Location: `web/` folder in the same repository

## Implementation Steps

### Step 1: Project Scaffold
- Initialize Vite + React + TypeScript project in `web/`
- Install dependencies: `jszip`, `file-saver`, `@types/file-saver`
- Set up CSS variables with the DeepDark theme palette
- Create global styles (reset, fonts, dark background)

### Step 2: Types & Data
Create in `src/types/index.ts`:
```ts
enum FileStatus { EmptyRequired, EmptyOptional, Filled }

interface DocumentOption {
  description: string
  path: string       // e.g. "Contabil/Folha de Pagamento/Contra Cheque"
  required: boolean
}

interface FileEntry {
  option: DocumentOption
  file: File | null   // browser File object (holds the bytes in memory)
  status: FileStatus  // derived from file + option.required
}
```

Create in `src/data/documentOptions.ts`:
- Port all 17 document definitions from MainViewModel constructor
- Organized by category (Contabil, Fiscal, Root)

### Step 3: Core Hook — `useDocumentManager`
Custom hook managing all application state:
- `files: FileEntry[]` — the document list with selected files
- `currentIndex: number | null` — currently selected item for preview
- `selectedMonth: number` — 1-12, defaults to current month
- `selectFile(index, file)` — attach a File to an entry
- `clearFile(index)` — reset an entry
- `exportZip()` — build ZIP using JSZip and trigger download via file-saver

Export logic:
1. Iterate files with non-null `file`
2. For each, get the file extension and build the path: `{option.path}{extension}`
3. Add to JSZip with folder structure
4. Generate blob and trigger download as `{MonthAbbrev}.zip`

### Step 4: UI Components (using `react-specialist` skill)

Invoke the **`react-specialist`** skill to build all React components. The skill will:
1. Read `.claude/references/REACT-REFERENCE.md` for project-specific patterns
2. Analyze the project structure and dependencies
3. Create components following React best practices with TypeScript

Provide the skill with the component specs, DeepDark theme palette, and TypeScript interfaces defined above.

Components to build:

**App.tsx** — Main layout (two-column grid)
- MenuBar at top
- FileList on the left (fixed ~500px width)
- FilePreview + Toolbar on the right

**MenuBar** — Simple bar with "Export ZIP" and month selector
- Maps to the WPF menu (simplified, no "Generate Folder" since web-only)

**FileList** — Scrollable list of FileListItem components

**FileListItem** — Single document row:
- Left border color based on status (green/red/orange)
- Document description label
- File name display (or empty)
- "Choose File" button → triggers hidden `<input type="file">`
- Right-click or button to "Clear" the file

**FilePreview** — Right panel showing selected file:
- PDF files: render in `<iframe>` using `URL.createObjectURL()`
- Images: render in `<img>` tag
- XML/OFX/text: read and display as text with `FileReader`
- Other: show file name and info

**Toolbar** — Bottom bar:
- Month dropdown (January–December)
- "Export as ZIP" button

## Feature Mapping (WPF → React)

| WPF Feature | React Equivalent |
|---|---|
| ObservableCollection<FileViewModel> | `useState<FileEntry[]>` |
| ICommand (ExportZip) | `onClick` handler / callback |
| OpenFileBehavior | Hidden `<input type="file">` |
| WebView2 preview | `<iframe>` / `<img>` / text display |
| BorderColorConverter | CSS class or inline style based on status |
| ComboBox (month) | `<select>` element |
| File.Copy to folder | JSZip folder structure |
| ZipFile.CreateFromDirectory | `JSZip.generateAsync("blob")` |
| ThemesController + DeepDark | CSS custom properties |

## Out of Scope
- AI-based file classification (future phase)
- Multiple theme switching (ship with DeepDark only)
- "Export as Folder" (browser limitation, ZIP only)
- Backend / server
